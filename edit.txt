در این حالت از React Router استفاده شده است، که به صورت Client-Side Routing عمل می‌کند. به عبارت دیگر:

آیا رفرش داریم؟

خیر، وقتی بین صفحات (Route) جا‌به‌جا می‌شوید، رفرش کامل صفحه اتفاق نمی‌افتد.

دلایل:

1. React Router از تاریخچه مرورگر (History API) استفاده می‌کند تا مسیر (URL) را تغییر دهد، بدون اینکه صفحه از ابتدا بارگذاری شود.


2. کامپوننت‌ها فقط محتوای خاص همان مسیر را تغییر می‌دهند (در اینجا محتوای children در DashboardLayoutBasic).


3. منابع ثابت (مثل CSS و JavaScript) دوباره درخواست نمی‌شوند.



سناریوهای بدون رفرش:

وقتی روی لینک‌های NAVIGATION کلیک می‌کنید، فقط محتوای مربوط به مسیر تغییر می‌کند.

ساختار اصلی، مانند تم یا منوهای جانبی، ثابت باقی می‌ماند.


نکته مهم:

اگر از Link یا useNavigate در React Router استفاده شود، انتقال بین صفحات بهینه و بدون رفرش کامل انجام می‌شود. در کد شما از AppProvider و Routes استفاده شده که به طور پیش‌فرض این رفتار را حفظ می‌کند.

تست بدون رفرش:

1. در مرورگر DevTools، به تب Network بروید.


2. روی یکی از لینک‌های ناوبری کلیک کنید.


3. مشاهده می‌کنید که فقط API‌های لازم یا منابع جدید (اگر وجود داشته باشند) درخواست می‌شوند و صفحه کامل رفرش نمی‌شود.
